1 From Monolithic to Microservice Architecture
1.1 Monolithic Architecture
A monolithic system integrates all modules into one deployable unit. Characteristics:
Âˆ Single codebase
Âˆ Single database
Âˆ Tight coupling
Âˆ Single deployment pipeline
1.2 Limitations of Monolith
1. Hard to scale specific modules
2. Risky deployment
3. Technology lock-in
4. Difficult team parallelization
1.3 Microservice Architecture
Microservice architecture decomposes the system into independent services. Key prop-
erties:
Âˆ Independent deployment
Âˆ Database per service
Âˆ REST or event-driven communication
Âˆ Independent scaling
Âˆ Fault isolation
2 Design Principles of Microservices
1. Single Responsibility (Service Level)
Each service owns exactly one business capability.
3

--- PAGE 4 ---
2. Database per Service
No shared database between services.
3. API-First Design
Define REST contracts before implementation.
4. Loose Coupling
Services interact only via APIs or events.
5. Stateless Services
Use JWT tokens for authentication.
6. Resilience
Apply:
Âˆ Circuit Breaker
Âˆ Retry
Âˆ Timeout
